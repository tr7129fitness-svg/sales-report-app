# 営業結果レポート

無料公開とランニングコストなしを前提にした、GitHub Pages + Firebase Firestore のWebアプリです。

## 使い方

ホームでは社員別のカレンダー確認と検索ができ、`報告入力` ボタンから入力画面へ移動できます。

Firebase Firestoreに共有保存されるため、同じ公開URLを見る社員同士で同じ報告データを確認できます。

## 現在の入力項目

- 日付
- 営業担当：谷本、會川、上原、ベイ
- 会社名
- 担当者名
- 料金内容：各項目ごとに `なし` または500円単位で選択
- 営業の所感

## 検索

ホームの検索欄では、会社名、担当者名、営業担当、日付、料金内容、営業の所感をまとめて検索できます。社員フィルターと検索は同時に使えます。検索中は該当した報告が箇条書きの一覧でも表示されます。

## プッシュ通知

iPhoneでSafariからホーム画面に追加したWebアプリに、報告追加時のWeb Push通知を送れる構成を追加しています。

通知を有効化するには、FirebaseプロジェクトをBlazeプランに変更し、Web Push用のVAPIDキーを発行して設定してください。

1. VAPIDキーを生成
2. `firebase-config.js` の `webPushPublicKey` に公開鍵を設定
3. Firebase Functionsの環境変数に `WEB_PUSH_PUBLIC_KEY` と `WEB_PUSH_PRIVATE_KEY` を設定
4. Firestore rulesとFunctionsをデプロイ
5. iPhoneのSafariで公開URLをホーム画面に追加
6. ホーム画面から開き、`通知を許可` を押す

通知は `reports` コレクションに新しい報告が追加された時だけ送信されます。編集では通知しません。

## 無料運用の注意

プッシュ通知を使わない場合は、無料枠内で運用するためFirebaseのSparkプランからアップグレードしないでください。

プッシュ通知を使う場合はCloud Functionsが必要になるため、FirebaseのBlazeプランが必要です。通常の営業報告件数であれば無料枠内に収まる見込みですが、Google Cloud側で予算アラートを設定してください。
